        How to Customize Vim

        franklinqin0 2020-01-02 tech

        Write our own .vimrc!

        Following the last blog about how to use Vim w/o any customization or plugin, this blog talks about how to customize Vim for better personal usage by writing a .vimrc.

        In Unix systems, we could create a hidden file (starting with .) vim ~/.vimrc for Vim or vim ~/.config/nvim/init.vim. In Windows, we set value of environmental variable $MYVIMRC and vim $MYVIMRC.

        In .vimrc, we don't need :.

        There are basically 4 parts in a .vimrc:

        1. Settings such as line numbers and colorscheme. To see what can be set, search by :h option-list
        2. Mappings such as jj for <Esc>
        3. Plugins' installations and settings
        4. Vimscript defines functions to streamline common tasks such as setting a title in a Python file

        # Mappings

        To map some key, say, dd to \, we could do map \ dd. We could then \ to delete the current line.

        To map in only one mode, we could do:

        • nm[ap]: map in normal mode
        • im[ap]: map in insert mode
        • vm[ap]: map in visual mode
        • cm[ap]: map in command mode

        The following enables <C-d> to delete a line under both normal and insert modes:

        :nmap \ dd
:imap \ <Esc>ddi
        1
        2

        To unmap dd from \, do unmap \.

        If we define:

        map \ dd
map - \
        1
        2

        Because Vim mapping is recursive by default, we could then - to delete the current line.

        Because some plugins define new mappings, there are cases when recursive mappings bring in accidental conflicts. Thus, we should always use non-recursive over recursive mappings.

        To create a non-recursive mapping under normal mode, do nnoremap <space> zz, after which pressing Space centers current line on the screen.

        To summarize, these are non-recursive mappings:

        • no[remap]: map in all modes
        • nn[oremap]: map in normal mode
        • ino[remap]: map in insert mode
        • vn[oremap]: map in visual mode
        • cno[remap]: map in command mode

        # Leader Key (See explanation from Learn Vimscript the Hard Way)

        After we define a <leader> key, we could define mappings combining the leader key:

        let mapleader=','
inoremap <leader>w <Esc>:w<cr>
        1
        2

        The leader key is , and we could now ,w to save.

        p.s.: <cr> is carriage return (<Enter> or <Return>).

        # Plugins

        We can use a plugin manager to manage plugins which would make Vim more customizable and powerful.

        # Plugin Manager

        There are many plugin managers for Vim/NeoVim, I switched from singled-threaded Vundle to multi-threaded vim-plug since installing and updating plugins in parallel save much time.

        # Plugin installation

        To add a plugin using vim-plug, just put the Github repo after Plug, between call begin and end:

        call plug#begin('~/.vim/plugged')
Plug 'mhinz/vim-startify'
call plug#end()

        # Useful Plugins

        Below are some useful plugins, which you can learn how to use through these videos in Chinese, searching on Google, or reading corresponding documentation:

        • start nvim with a fancy screen: vim-startify
        • customize your colorscheme to your favorite: colo[rscheme] {your favorite}. Personally I liked: gruvbox, hybrid, solarized, molokai, etc.
        • beautify status and tab lines: vim-airline
        • display indentation levels with thin vertical lines: indentline
        • file explorer as tree: nerdtree
        • fuzzy file finder: 1. fzf is a really good command-line fuzzy finder, and in Vim you could use fzf.vim. 2. more traditional and is less powerful: ctrip.vim. I prefer fzf b/c of its blazing speed.
        • jump to any character instantly on screen: vim-easymotion
        • overview of file structure: majutsushi/tagbar
        • change, delete, or add quotes/parentheses: vim-surround
        • find and replace in multiple files: far.vim
        • highlight different words simultaneously: lfv89/vim-interestingwords
        • code completion: 1. neoclide/coc.nvim is a new tool by a Chinese frontend pro but needs to install nodejs before installation. 2. Shougo/deoplete.nvim does a similar job but I feel the user experience of coc is better
        • format code: sbdchd/neoformat
        • lint code: w0rp/ale
        • comment code: tpope/vim-commentary
        • easily align code: junegunn/vim-easy-align
        • easier look on parentheses: luochen1990/rainbow

        These are great plugings to combine Git and Vim:

        • the best Git wrapper in Vim: tpope/vim-fugitive could do the following commands: :G[status], :Gblame, :Gedit for a try
        • git diff in the gutter (sign column): airblade/vim-gitgutter
        • git commit browser: junegunn/gv.vim
